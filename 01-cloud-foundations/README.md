### Key Takeaways - AWS Shared Responsibility Model

**Simple Analogy**:  
Think of AWS as a **landlord** and you as a **tenant** renting an apartment.

- **AWS (Landlord)** is responsible for **Security OF the Cloud**  
  → They protect the building, structure, electricity, water pipes, and common areas.

- **Customer (You)** is responsible for **Security IN the Cloud**  
  → You protect everything inside your own apartment.

### Activity Explanation (Simple Average Joe Style)

1. **OS patches on EC2** → **Customer** (You fix things inside your apartment)
2. **Physical security of data center** → **AWS** (Landlord guards the whole building)
3. **Virtualization infrastructure** → **AWS** (Landlord manages the building technology)
4. **EC2 Security Group settings** → **Customer** (You decide who can enter your apartment)
5. **Applications configuration on EC2** → **Customer** (You arrange furniture inside)
6. **Oracle patches on RDS** → **AWS** (Managed service - landlord takes care of it)
7. **Oracle patches on EC2** → **Customer** (You installed it yourself, so you maintain it)
8. **S3 bucket access configuration** → **Customer** (You control who can use your storage room)
